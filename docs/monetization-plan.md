# CARENOTE 수익 모델 기획서

> 핵심 원칙: **기입자(환자 본인)는 무료, 관리자(보호자)는 유료.**
> 지불 의사가 높은 쪽(걱정하는 가족)이 지불하고, 실제 사용자(어르신·아이)는 진입장벽 0.

---

## 1. 보호자 연동 모델 (Family Care) — 유료 핵심

### 1-1. 개념
| 역할 | 설명 | 과금 |
|---|---|---|
| **기입자(피보호자)** | 환자 본인 — 복약·혈당 기록, 자동수집 | **무료** |
| **관리자(보호자)** | 자녀·부모 등 — 피보호자를 원격 모니터링 | **유료 구독** |

- **양방향 지원**
  - 자녀 → 부모: 부모님 복약·혈당 원격 확인 (고령 당뇨/고혈압)
  - 부모 → 아이: 아이가 약 잘 먹는지 확인 (소아 천식·당뇨·ADHD 등)
- 관리자 1명이 **여러 피보호자** 관리 (부모+장인장모, 자녀 여러 명)

### 1-2. 유료(관리자) 기능
1. **원격 대시보드** — 피보호자의 복약 여부·혈당·혈압·활동을 실시간 확인
2. **복약 누락 원격 알림** — 정해진 복약 시간이 지나도 미복용이면 **관리자에게** 알림
   - 앱 푸시(무료) → 카카오 알림톡/문자(유료 원가, 상위 요금제)
3. **이상 수치 알림** — 저혈당/고혈당·고혈압 등 위험 수치 발생 시 즉시 통지
4. **건강 리포트** — 주간·월간 요약, **병원 제출용 PDF**
5. **다중 피보호자 관리** — 한 화면에서 여러 가족
6. **메모·응급 연락 공유** — 가족 간 상태 메모

### 1-3. 무료(기입자) 기능
- 모든 기록·자동수집(HealthKit·기기연동)·본인 복약 알림·기본 대시보드
- 즉, **환자 경험은 완전 무료** — 이게 사용자 확산의 핵심

### 1-4. 알림 발송 채널 (원가 구조)
| 채널 | 원가 | 배치 |
|---|---|---|
| 앱 푸시 | 무료 | 기본 (무료 요금제도 포함 가능) |
| 카카오 알림톡 | 건당 과금(사업자·발신프로필 필요) | 유료 상위 요금제 |
| 문자(SMS) | 건당 과금(알리고/NHN 등) | 유료 상위 요금제 |

> 카톡·문자는 **발신 원가**가 있으므로 유료 기능에 배치해 원가를 회수. 앱 푸시는 무료로 폭넓게.

### 1-5. 가격(안)
- **케어 구독**: 월 **₩4,900~6,900** (피보호자 1명 포함)
- 추가 피보호자 **+₩2,000/명**
- 연 구독 시 2개월 할인
- 상위 요금제(카톡·문자 알림 포함): 월 **₩9,900**

## 2. 광고 리워드 수익 (무료 사용자)

- **보상형 광고(리워드)**: 무료 사용자가 자발적으로 광고 시청 → 보상
  - 보상 예: 프리미엄 리포트 1회 열람, AI 분석 추가 횟수, 기부 포인트, 배지
- 구현: Google AdMob 보상형 광고
- ⚠️ **원칙**: 의료·시니어 앱은 광고 과하면 신뢰↓ → **절제된 배치**, 강제 노출 최소화, 리워드는 "자발적 시청"만

---

## 3. 커머스 제휴 — 쿠팡 파트너스 + 자체 광고

### 3-1. 쿠팡 파트너스 (제휴 커미션)
- 건강 상품 추천 → 제휴 링크 구매 시 커미션
  - 혈당기·혈압계·리브레 소모품, 영양제, 시니어 용품, 저당 식품 등
- 배치: "추천 상품" 섹션, 리포트 하단, 관련 화면(맥락형)
- ⚠️ **고지 의무**: "이 포스팅/화면은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다." 문구 필수

### 3-2. 자체 광고제안 페이지 (직접 배너 판매)
- **"광고·제휴 문의"** 랜딩 페이지 별도 운영
- 대상: 건강기기 제조사, 제약/건기식, 보험사, 요양·실버 서비스
- 상품: 앱 내 배너, 리포트 스폰서, 뉴스레터 등
- 단가표·매체소개서(사용자 수·연령대·건강관심사) 준비

### 3-3. 의료광고 주의
- 건기식·의료기기 광고는 **표시광고 심의/규정** 확인 필요
- 과장·효능 오인 표현 금지, 광고임을 명확히 표기

## 4. 기술 설계 (구현 밑그림)

### 4-1. 데이터 모델
```
cn_guardianship (보호 관계)
  id, guardian_id(관리자), ward_id(피보호자),
  status('pending'|'active'|'revoked'),
  permissions(jsonb: 복약·혈당·리포트 열람 범위),
  created_at
```
- **연결 플로우**: 관리자가 피보호자를 초대(코드/이메일) → 피보호자가 **수락(명시적 동의)** → active
- 민감정보(건강)라 **피보호자 동의 필수** (개인정보보호법)

### 4-2. RLS 확장
- `diabetes_data` SELECT 정책에 "보호자 관계면 허용" 조건 추가:
  ```sql
  using (
    user_id::text = (select auth.uid())::text
    OR exists (select 1 from cn_guardianship g
               where g.ward_id = diabetes_data.user_id
                 and g.guardian_id::text = (select auth.uid())::text
                 and g.status = 'active')
  )
  ```
- 보호자는 **읽기 전용**, 쓰기는 불가

### 4-3. 알림 엔진
- **Edge Function + 스케줄(cron)**: 복약 예정 시간 + 유예(예 30분) 후에도 `meds[id]` 미체크 → 관리자에게 발송
- 채널: 앱 푸시 → (상위요금제) 알림톡/문자 API 호출

### 4-4. 결제
- iOS 디지털 구독은 **Apple 인앱결제(IAP) 필수** → RevenueCat 권장
- `cn_profiles.plan`('free'|'care'|'care_plus') + 구독 상태 동기화

## 5. 우선순위 로드맵

| 단계 | 내용 |
|---|---|
| **P1** | 보호 관계(cn_guardianship) + 연결/수락 플로우 + RLS 확장 |
| **P2** | 관리자 원격 대시보드 (피보호자 복약·혈당 조회) |
| **P3** | 복약 누락 알림(앱 푸시) — 무료 범위로 먼저 |
| **P4** | IAP 구독(RevenueCat) + 유료 게이팅 |
| **P5** | 카톡·문자 알림(상위 요금제), 리포트 PDF |
| **P6** | 쿠팡 파트너스, 광고제안 페이지, 리워드 광고 |

> 추천: **P1~P3(가족 연결 + 원격확인 + 푸시알림)** 을 먼저 무료로 완성해 가치를 증명하고,
> 그다음 **P4(결제)** 로 유료화. 알림톡/광고는 후순위.
